## L1: 算法分析基础 (§2.1–2.3 + §2.4.1-2.4.2)

### Big-Oh 记号体系

| 记号 | 含义 | 类比 |
|------|------|------|
| O(f(N)) | 上界，增长率 ≤ f(N) | ≤ |
| Ω(g(N)) | 下界，增长率 ≥ g(N) | ≥ |
| Θ(h(N)) | 紧确界，增长率 = h(N) | = |
| o(p(N)) | 严格上界，增长率 < p(N) | < |

**关键**：所有记号都是渐进（asymptotic）的——只关心 N 足够大时的行为，忽略常数因子和低阶项。

### 三条核心规则 (Weiss p.52-53)

1. **加法**：T₁(N) + T₂(N) = O(max(f(N), g(N)))
2. **乘法**：T₁(N) × T₂(N) = O(f(N) × g(N))
3. **多项式**：k 次多项式 = Θ(Nᵏ)

### 常见增长速率（从慢到快）

c < log N < log² N < N < N log N < N² < N³ < 2ᴺ

### 分析规则 (Weiss §2.4.2)

1. **FOR 循环**：循环体时间 × 迭代次数
2. **嵌套循环**：从内向外分析，乘积
3. **顺序语句**：相加，取最大项
4. **If/Else**：测试 + max(分支1, 分支2)

### 递归 Fibonacci 的教训

```cpp
// 灾难：O(φᴺ) ≈ O(1.618ᴺ)
long fib(int n) {
    if (n <= 1) return 1;
    return fib(n-1) + fib(n-2);
}
```

递推关系：T(N) = T(N-1) + T(N-2) + O(1)

令 g_N = T(N) + 1，则 g_N = g_{N-1} + g_{N-2}（标准 Fibonacci！）

→ T(N) = 2·F_{N+1} - 1 = O(φᴺ)

**核心教训**："Don't compute anything more than once." — Weiss p.60

### 分析习惯

- 先算精确执行次数，再丢弃低阶项和常数
- 嵌套循环从内向外分析，注意内层循环的边界是否依赖于外层变量
- 条件判断也是工作，要计入复杂度
- 最坏情况分析是默认选择——它提供对所有输入的保证

## L2: 最大子序列和 + 对数时间 (§2.4.3–2.4.5)

### 最大子序列和问题的四种算法

| 算法 | 复杂度 | 核心思想 |
|------|--------|----------|
| maxSubSum1 | O(N³) | 暴力枚举所有起点+终点+累加 |
| maxSubSum2 | O(N²) | 消除最内层：Σ(j,i) A_k = A_j + Σ(j-1,i) A_k |
| maxSubSum3 | O(N log N) | 分治：左半/右半/跨越中点三者取最大 |
| maxSubSum4 | O(N) | Kadane：负前缀不可能为最优解前缀，丢弃归零 |

**算法 4 是在线算法（online algorithm）**：只需一次扫描，随时可给出当前答案。

### 分治法递推分析

T(N) = 2T(N/2) + O(N)，当 N = 2ᵏ 时 T(N) = N·(k+1) = O(N log N)

### 识别对数时间的规则 (Weiss p.66)

> O(log N): 每次用 O(1) 时间将问题规模**缩小一个分数**（通常 1/2）
> O(N): 每次用 O(1) 时间将问题规模**减少一个常数**（如减 1）

### 三种 O(log N) 算法

| 算法 | 关键机制 | 最坏情况 |
|------|----------|----------|
| **二分查找** | 每次比较将搜索范围减半 | ⌈log(N−1)⌉+2 次迭代 |
| **欧几里得 GCD** | 每两次迭代余数至少减半 (Theorem 2.1) | 连续 Fibonacci 数 ~1.44 log N |
| **快速幂** | 指数每步减半，至多 2 次乘法/步 | 2 log N 次乘法 |

### 快速幂的三种错误 (Weiss p.70)

```cpp
// 8a, 8b: 无限循环（n=2 时递归调用自身，无进展）
pow(pow(x, 2), n / 2);     // 8a
pow(pow(x, n / 2), 2);     // 8b

// 8c: 退化到 O(N)（两次递归调用）
pow(x, n / 2) * pow(x, n / 2);  // 8c
// T(N) = 2T(N/2) + O(1) → O(N)
```

### 实践收获

- `mid = left + (right - left) / 2` 防止整数溢出
- 递归版需要包装函数封装 `left=0, right=size-1`
- Benchmark 时对大 N 的 O(N³) 算法必须加保护跳过

## L3: 习题实战心得 (2026-06-12)

### 2.15 — 二分查找变种：排序数组中找 Aᵢ = i

**问题**：严格递增数组，找出是否存在下标 i 使得 Aᵢ = i。

**关键洞察**：定义 `f(i) = Aᵢ - i`。由严格递增性：
```
f(i+1) - f(i) = A_{i+1} - A_i - 1 ≥ 0
```
→ f 非递减！问题转化为在 f 中二分查找 0。

**心得**：这题的难点在于**"看不见 target"**——没有显式的查找目标。技巧是**构造辅助函数并证明其单调性**，然后再套二分模板。不是"改了二分"，而是"改了对问题的编码方式"。

### 2.17a — 最小子列和（Kadane 翻转）

**问题**：求最小子列和。

**核心**：把 Kadane 最大值算法的逻辑翻转：
- 最大值中：**负前缀**不可能为最优前缀（丢弃归零）
- 最小值中：**正前缀**不可能为最优前缀（丢弃归零）

```cpp
if (thisSum > 0) thisSum = 0;  // 正前缀只会让最小值变大
```

**陷阱**：初始化和空子列——minSum 初始化为 0，允许空子列和为答案。

### 2.17b — 最小正子列和（O(N log N)）

**为什么 Kadane 不行**："正"的约束破坏了单调性。一个负前缀后面跟大正数，可能产生很小的正和——不能轻易丢弃。

**O(N log N) 解法**：
1. 子列和 = `prefix[j] - prefix[i]`（i < j）
2. 对每个 j，找**最大的 prefix[i] < prefix[j]**（使差值为正且最小）
3. `std::set` + `lower_bound` → 找前驱，O(log N) 每次
4. 总 O(N log N)，空间 O(N)

**心得**：前缀和 + 有序查找是处理"带约束子列和"的通用武器。另外，`std::set` 不能存重复值——对本题不影响（因为重复前缀意味着差值为 0，不符合"正"的条件）。

### 2.17c — 最大子列积（O(N)）

**与和的本质区别**：**负负得正**。一个很小的负数积，乘负数可能翻身变最大。

**核心技巧**：同时追踪 `curMax` 和 `curMin`：
```
遇到负数：swap(curMax, curMin)  // 负号翻转大小关系
curMax = max(x, curMax * x)     // 另起炉灶 vs 延长
curMin = min(x, curMin * x)
```

**设计选择**：空子列积 = 1（类比教材允许空子列和为 0）→ 对全负/全零数组，答案为 1（空子列）。若不接受，应初始化为 `v[0]` 并从 i=1 开始循环。

### 2.31 — 二分查找的 off-by-one 死循环

**问题**：迭代版 `low = mid`（而非 `low = mid + 1`），算法还能工作吗？

**答案**：不能。整数除法向下取整，当区间 ≤ 2 时 `mid = low`。`low = mid` 不缩小范围 → 死循环。`mid + 1` 保证每次至少排除一个元素，区间严格收敛。

**心得**：这是二分查找最经典的 bug。不靠记忆，要真正理解"为什么是 +1 而不是 mid"——`mid` 可能等于 `low`，所以只靠 `mid` 无法保证前进。
