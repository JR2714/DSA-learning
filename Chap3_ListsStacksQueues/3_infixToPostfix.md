# 3.3 分析：中缀转后缀的优先级规则
    在 infix-to-postfix 算法中，当我们遇到一个操作符时，要弹出栈中优先级 ≥ 当前操作符的符号（遇到 ( 停止）。为什么是 ≥ 而不是 >？考虑表达式 a - b - c。如果使用 > 会发生什么？结果会是什么形式，是否保持了正确的运算顺序？请用一两句话解释。

----

对于给定 infix 式 $a-b-c$ ，当弹出优先级 “$>$” 当前操作符时，有以下步骤：

|步骤|stack|output|
|----|-----|------|
|1|$-$|a|
|2|$-$ $-$|a b|
|3||a b c $-$ $-$|

运算顺序错误，这破坏了日常运算中的“左边先算”的规则，会导致本来应该算$(a-b)-c$，变为$a-(b-c)=a-b+c$从而导致错误。
