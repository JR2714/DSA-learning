Jacob wants rigorous code review on every exercise submission, covering:
- Correct syntax and language usage
- Edge case handling (empty input, boundary values, overflow, etc.)
- Good programming style (naming, formatting, comments)
- Code efficiency (avoid unnecessary copies, correct algorithm)
- Stability and robustness (no undefined behavior, memory safety)

**Why:** Jacob wants to build good coding habits from the start of his DSA learning.
**How to apply:** On every code review, systematically check all of the above dimensions. Be strict but constructive — point out every issue, no matter how small. If code is perfect, say so explicitly.
